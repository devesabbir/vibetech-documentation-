Prerequisites

React is constructed leveraging Node.js. as its foundation. To initiate your journey with React, it's essential to have an up-to-date version of Node.js and For managing routing within your React application, the employment of React Router DOM is necessary.

Installing Development Environment

We require many packages (dependencies) to run our site. Move into /vibetech-react (project root) directory and run command below. 

npm install

After the installation, you should have a folder named /node_modules in the project root directory..

How To Run Development Server

To start our development server run command below:

npm run dev

Open your browser and visit http://localhost:5173!. You should see a page like below.

Voila! You are a genious. Now open the code editor and start hacking!

CLI Commands

Some useful commands to work with the project.

2. npm run build for Generating production build

3. npm run preview To serve build files at http://localhost:4173

File Structure

Follow the steps below to setup your site template:

  1. Unzip the downloaded package and open the /vibetech-react folder to find all the template files. You will need to upload these files to your hosting web server using FTP or localhost in order to use it on your website.
  2. Below is the folder structure and needs to be uploaded to your website or localhost root directory:
    • vibetech-react/src/style - Sass files
    • vibetech-react/src/pages - All pages
    • vibetech-react/src/routes - Routes files
    • vibetech-react/public/image - Images files
    • vibetech-react/index.html - Root file
  3. You should upload all or specific React files as per your need.
  4. You are good to go for adding your content now!

Props

isSticky The isSticky prop essentially controls how the Header component behaves during scrolling. When isSticky is set to true, the header exhibits a sticky behavior, staying fixed at the top as you scroll down. Conversely, setting isSticky to false eliminates the sticky behavior, allowing the header to remain in a standard, non-scrolling position.


styleVariant The styleVariant prop in the header component acts as a specifier for the header variant. Four distinct styles are designed for headers, each represented by a value: 'one', 'two', 'three' and 'four' By specifying one of these values, you can choose from the available header styles. If the value is false then the header is free of style and ready to receive styles from the custom class.

logoVarient The logoVarient prop serves as a mechanism for obtaining logo variant information. This prop offers three variation: 'LIGHT', 'DARK', and 'WHITE', each triggering a dynamic update of the alternate logo based on the selected variant.


className The className prop in the Header component serves as a versatile styling option. It eagerly accepts any class name you provide, giving you the flexibility to define and apply styles to the header tag as needed.


button The button prop in the Header component is crafted to receive an object providing information for the header's contact button. This object must include: text The text displayed on the button. path The route path navigated when the button is clicked. classNames An optional property that accommodates any class names for custom styling of the button.


menu Configure the alignment and styling of the menu in the Header component using the menu prop. This prop expects an object with two properties: align Determines the alignment with values 'start', 'end', 'center' or 'between' The className an optional property for including any class names, allowing customization of the menu's appearance.

Here's a example of how to use the <SectionTitle/> component in Vibetech React Template:



Props

variant (Type: "one" | "two"): Specifies the title variant Use "one" for a regular heading. Use "two" for a stroked heading.

strokeVariant (Type: "one" | "two"): This prop works only when the variant is set to "two". Specifies the strokeVariant for the stroked heading.

className (Type: String): The className prop is Optional class name for the heading. Allows custom styling for the heading element.

headingTitle (Type: { text, className }): This Object representing the main title. Includes a text property for the title content. Optional className property for custom styling.

heading (Type: { text, className }): This Object representing a sub-heading. Includes a text property for the sub-heading content. Optional className property for custom styling.

headingDescription (Type: { text, className }): This Object representing a description under the heading. Includes a text property for the description content. Optional className property for custom styling. Note: This prop is applicable only when the variant is set to "one" (for regular headings).
